Co i dlaczego wyświetli się w konsoli po uruchomieniu poniższego programu?

class TextModification {
    public static void main(String[] args) {
        String original = "Hello";
        duplicateText(original);
        System.out.println(original);
    }

private static String duplicateText(String source) {
        source = source + source;
        return source;
    }
}

prawidłowa odpowiedź to "Hello", czyli wywołanie metody duplicateText() nie ma żadnego wpływu na literał przypisany do zmiennej original. Obiekty w Javie przekazywane są do metod poprzez kopie referencji.

Należy pamiętać, że obiekty typu String są niemodyfikowalne (immutable), a tym samym w pamięci powstaje nowy obiekt "HelloHello". Na obiekt ten będzie wskazywała jednak tylko referencja source, referencja original wskazuje nadal na ten sam obiekt, na który wskazywała wcześniej.


Ile obiektów typu String powstanie po uruchomieniu poniższego kodu?

String x = new String("Hello");
String y = x;
String z = "Hello";

Powstanie dwa obiekty! W Pierwszej linii i tyle. Jeden z konstruktora drugi z literału i on zostanie zapisany do String pool Pozostałe wskazują na niego.
