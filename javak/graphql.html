GraphQl jest alternatywą dla REST.


W odróżnieniu do REST w GraphQl jest tylko jeden endpoint na który strzelamy.


Są dwa podejscia tworzenia aplikacji:


  1. Tworzmąc reselvery, schema i Query/mutations
  2. Przy wykorzystaniu adnotacji @

Tworzenie resolvers/schema/query/mutations

3 kluczowe elementy GraphQL to :

  • query i mutation
  • resolvery
  • schema
Query służy do pobierania danych, Mutation do modyfikowania.
Resolvery są tym samym co RestController czyli definiuje się tam metody które wykonują całą logikę na danych
Schema Opisuje strukturę. Określa ona strukturę w jaki sposób będzie przedstawiana dana odpowiedź, czyli zwracany JSON.
Schema opisuje przechowywane typy oraz to jak będziemy te dane przekazywać.
Podobnie jak REST działamy na bazie danych może być H2 i na jakimś modelu np. Movie.

Do Testowania najwygodniej jest wykorzystać narzędzie GraphiQL

http://localhost:8080/graphiql

Można też użyć Postmana.

Example

query{
getMovie(title: "John Wick"){
title
year
director
rate
}
}

query{
allMovies{
title
year
director
rate
}
}
